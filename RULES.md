# GitHub 운영 규칙

이 저장소는 스터디 발표 자료와 질문 내용을 관리하기 위한 저장소입니다.
모든 파일 업로드와 반영은 아래 규칙에 따라 진행합니다.

---

## 1. 기본 원칙

`main` 브랜치에는 직접 파일을 올리지 않습니다.

모든 사용자는 GitHub 웹의 **Add file** 기능을 이용해 파일을 업로드하고,
반드시 새 브랜치를 생성한 뒤 **Pull Request(PR)** 를 통해 반영해야 합니다.

전체 흐름은 다음과 같습니다.

```text
질문자: Add file로 파일 업로드
→ 질문자: 본인 이름으로 새 브랜치 생성
→ 질문자: main 브랜치로 PR 생성
→ 질문자: PR 번호 기준 Issue 생성
→ 발표자: 발표 후 Issue에 comment로 답변
→ 발표자: 답변 완료 후 PR merge
→ Issue 자동 close & 브랜치 자동 delete 확인
```

---

## 2. 진행 시점 규칙

### 질문자

질문자는 **발표 시작 전까지** 아래 작업을 완료해야 합니다.

```text
1. 본인이 공부한 내용 업로드
2. 본인 이름으로 새 브랜치 생성
3. main 브랜치로 PR 생성
4. PR 번호를 기준으로 Issue 생성
5. PR 본문에 연결된 Issue 번호 작성
```

### 발표자

발표자는 **발표가 끝난 후 스터디 진행 중에** 아래 작업을 완료해야 합니다.

```text
1. 본인 발표와 연결된 Issue 확인
2. Issue 댓글로 질문에 답변
3. 답변 완료 후 해당 질문자의 PR merge
4. Issue close 및 브랜치 delete 확인
```

---

## 3. 파일 업로드 규칙

파일을 업로드할 때는 다음 순서로 진행합니다.

1. 저장소에 접속합니다.
2. `Add file`을 클릭합니다.
3. `Upload files` 또는 `Create new file`을 선택합니다.
4. 파일을 추가합니다.
5. `main` 브랜치에 직접 커밋하지 않습니다.
6. 반드시 새 브랜치를 생성합니다.
7. 새 브랜치에서 PR을 생성합니다.

---

## 4. 브랜치 규칙

브랜치 이름은 본인 이름으로 작성합니다.

예시:

```text
minhyung
jiwon
```

PR이 merge되면 해당 브랜치는 자동 삭제됩니다.
삭제된 브랜치는 다시 사용하지 않고, 새로운 업로드가 필요할 때는 새 브랜치를 생성합니다.

---

## 5. Pull Request 규칙

파일을 업로드한 사람은 반드시 PR을 생성해야 합니다.

PR 제목은 아래 형식을 따릅니다.

```text
[주차] 이름 - 해당 주제
```

예시:

```text
[1주차] 이민형 - Docker 복습과 Hello, k8s
```

PR 본문에는 본인이 공부한 내용을 작성합니다.

또한 PR과 연결된 Issue를 자동으로 닫기 위해 PR 본문 하단에 아래 형식을 작성합니다.

```text
Closes #이슈번호
```

예시:

```text
Closes #12
```

---

## 6. Issue 질문 규칙

질문자는 PR을 생성한 뒤, 해당 PR 번호를 기준으로 Issue를 생성합니다.

Issue 제목은 아래 형식을 따릅니다.

```text
[질문] PR #번호 이름 - 해당 주제
```

예시:

```text
[질문] PR #7 이민형 - Docker 복습과 Hello, k8s
```

Issue 본문에는 발표자에게 물어볼 질문을 작성합니다.

```md
## 관련 PR

PR 번호: #

## 발표자

이름:

## 질문

1. 
2. 
```

질문은 발표자가 이해하고 답변할 수 있도록 구체적으로 작성합니다.

---

## 7. 발표자 답변 및 Merge 규칙

발표자는 발표가 끝난 후 스터디 진행 중에 Issue를 확인하고,
댓글로 질문에 답변해야 합니다.

답변이 완료되기 전까지는 해당 PR을 merge하지 않습니다.

PR을 merge하기 전에는 아래 내용을 확인합니다.

```md
- [ ] 질문자가 발표 시작 전에 PR을 생성했는가?
- [ ] 질문자가 발표 시작 전에 Issue를 생성했는가?
- [ ] PR이 main 브랜치가 아닌 새 브랜치에서 생성되었는가?
- [ ] PR 본문에 `Closes #이슈번호`가 작성되어 있는가?
- [ ] 발표자가 Issue 댓글로 질문에 답변했는가?
```

위 조건을 모두 만족하면 발표자가 PR을 merge합니다.

---

## 8. 금지 사항

아래 행동은 금지합니다.

```text
- main 브랜치에 직접 파일 업로드
- PR 없이 파일 반영
- 발표 시작 전까지 PR을 올리지 않는 것
- 발표 시작 전까지 Issue를 생성하지 않는 것
- Issue 질문에 답변하지 않은 상태에서 merge
- 관련 Issue 없이 PR merge
- 이미 merge된 브랜치 재사용
```

---

## 9. 요약

이 저장소의 핵심 규칙은 다음과 같습니다.

```text
main 직접 업로드 금지
질문자는 발표 시작 전 PR 생성
질문자는 발표 시작 전 Issue 생성
PR에는 본인이 공부한 내용 작성
Issue에는 발표자에게 할 질문 작성
발표자는 발표 종료 후 Issue 댓글로 답변
답변 완료 후 발표자가 PR merge
merge 후 Issue close & 브랜치 delete 확인
```
